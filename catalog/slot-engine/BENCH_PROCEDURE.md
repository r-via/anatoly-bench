## Fixture parameters

- **Project path**: `catalog/slot-engine/project` (substitute for `<fixture>`
  in the shared doc's commands → `slot-engine`).
- **Scored axes**: 5 — `correction`, `utility`, `duplication`,
  `overengineering`, `best-practices`. The `tests` and `documentation` axes
  are excluded (`scored_axes` config + `.anatoly.yml enabled:false`): the
  project carries neither tests nor JSDoc, so precision on those axes would
  be mechanically destroyed.
- **Ground truth**: [SPEC.md](./SPEC.md) (YAML catalog, never enters
  `project/`, never seen by Anatoly).
- **Scorer**: deterministic bipartite match on `(file, line ± tolerance,
  axis, verdict)`.

## Scoring

After copying the run back (see shared doc), score on the local machine:

```bash
cd ~/projects/anatoly-bench
npm run build  # only if the bench source changed

VERSION=v18   # increment from the last baseline
DATE=$(date +%Y-%m-%d)
SPEC=catalog/slot-engine/SPEC.md
RUN=catalog/slot-engine/project/.anatoly/runs/$RUN_ID

node dist/cli.js score \
  --spec $SPEC \
  --report $RUN \
  --json baselines/${DATE}_slot-engine-${VERSION}.json \
  --md   baselines/${DATE}_slot-engine-${VERSION}.md
```

The JSON score contains `global_f1`, `per_axis`, `misses` and
`false_positives`; the Markdown also surfaces run metadata (commit, duration,
cost, tokens).

## Comparing to previous baselines

```bash
for f in baselines/*-slot-engine-v*.json; do
  echo "$(basename $f): $(grep -oE '"global_f1":[^,]+' $f)"
done
```

## slot-engine-specific gotcha

- **The bipartite scorer requires a stable `project/` tree**. If `project/`
  is regenerated by Evolve, the cataloged `(file, line)` anchors in SPEC.md
  may drift — re-validate the catalog against the new tree before trusting a
  baseline.
