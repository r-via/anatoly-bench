# Bench procedure — train-dispatch specifics

> Shared infrastructure (VM/SSH/GPU, `make update`, launching/monitoring a
> run, artifact layout, `--no-cache` rationale, generic gotchas) ->
> [docs/04-bench-procedure.md](../../docs/04-bench-procedure.md). This file
> only covers what is specific to the `train-dispatch` fixture.

## What this fixture is for

`train-dispatch` is the **behavioural-invariant** companion to `slot-engine`.
Where slot-engine ships numeric/structural defects, train-dispatch ships
defects that only reveal themselves as violations of properties over an
execution: ordering (priority inversion), temporal separation (headway),
mutual exclusion (double block occupancy), liveness (single-track deadlock),
and conservation (a train dispatched twice), plus one trivial numeric-target
defect (dwell) whose effect is purely behavioural (lateness). It exists to
drive and measure a behavioural-invariant detector. The on-time numeric
objective derives from the behaviour, not the other way around.

## Fixture parameters

- **Project path**: `catalog/train-dispatch/project` (substitute for
  `<fixture>` in the shared doc's commands -> `train-dispatch`).
- **Scored axes**: 1 — `correction` only. Every other axis is excluded via
  `scored_axes` in the catalog (and should be disabled in `.anatoly.yml` the
  same way slot-engine disables `tests`/`documentation`). The project ships no
  tests and almost no JSDoc, and the four noise modules (`routing.ts`,
  `kpi.ts`, `telemetry.ts`, `format.ts`) are deliberately clean, so any finding
  on them is a precision error. Single-axis scope keeps the F1 signal free of
  the cross-axis noise that has historically masked a correction-only lever.
  Because `scored_axes` is a singleton, `global_f1` equals the correction-axis
  F1 directly.
- **Ground truth**: [SPEC.md](./SPEC.md) (YAML catalog, never enters
  `project/`, never seen by Anatoly). Six cataloged correction defects,
  ordered by increasing difficulty: INV-DWELL (trivial), INV-PRIORITY,
  INV-HEADWAY, INV-CONSERVATION (medium), INV-MUTEX, INV-DEADLOCK (hard).
- **Determinism**: the engine has no RNG. A single `runSchedule()` run is the
  ground truth — `verify-runtime.mts` reconstructs behaviour from the raw
  trace, so there is no Monte-Carlo and no flakiness.

## Validate the fixture BEFORE benching (Evolve convergence)

This fixture is authored the slot-engine way: the `src/` tree is generated by
Evolve to converge against `SPEC.md`, not hand-written. Before any scored run:

```bash
# on the VM, inside the project dir
cd ~/project/anatoly-bench/catalog/train-dispatch/project
evolve   # reads evolve.toml: check=../verify.sh, spec=../SPEC.md
```

Convergence means `verify.sh` exits 0: the code typechecks, the public API and
12-module layout are present, the source signatures hold, and the deterministic
simulation confirms all four operating guarantees are measurably violated
(on-time rate below the 85% floor, >= 1 double-occupancy tick, >= 2 stranded
trains, >= 1 duplicate arrival, >= 1 headway violation). If convergence stalls,
the most likely lever is the published `scheduledArrival` values in
`timetable.ts` and the on-time floor in `verify-runtime.mts` — the structural
checks (deadlock, double occupancy, duplicate arrival, headway) are independent
of the absolute schedule magnitudes.

After convergence, re-validate the catalog `(file, symbol)` anchors against the
generated tree (see the gotcha below) before trusting any baseline.

## Scoring

After copying the run back (see shared doc), score on the local machine:

```bash
cd ~/projects/anatoly-bench
npm run build  # only if the bench source changed

VERSION=v1   # increment from the last baseline
DATE=$(date +%Y-%m-%d)
SPEC=catalog/train-dispatch/SPEC.md
RUN=catalog/train-dispatch/project/.anatoly/runs/$RUN_ID

node dist/cli.js score \
  --spec $SPEC \
  --report $RUN \
  --json baselines/${DATE}_train-dispatch-${VERSION}.json \
  --md   baselines/${DATE}_train-dispatch-${VERSION}.md
```

The JSON score contains `global_f1`, `per_axis`, `misses` and
`false_positives`; the Markdown also surfaces run metadata (commit, duration,
cost, tokens). With a single scored axis, `global_f1 == per_axis.correction.f1`.

## train-dispatch-specific gotchas

- **The bipartite scorer requires a stable `project/` tree.** If `project/` is
  regenerated by Evolve, the cataloged `(file, line)` anchors in SPEC.md may
  drift. Most catalog entries anchor on `file` + `symbol` rather than a line
  hint, which is robust to regeneration, but re-validate after every Evolve run
  before trusting a baseline.
- **Two defects share `interlocking.ts`** (INV-MUTEX and INV-DEADLOCK) on
  different symbols. The scorer matches on `(file, symbol)`; if Anatoly reports
  only a file-level finding it can match at most one of the two. Watch the
  `misses` list to tell which behavioural invariant the detector is blind to.
- **INV-PRIORITY, INV-HEADWAY, INV-MUTEX, INV-CONSERVATION carry no line
  hint** (the catalog anchors them on `file` only, or `file` + `symbol` for
  DWELL). A finding anywhere in the named file with a compatible verdict
  matches; this is intentional, since behavioural defects are reasoned about at
  the routine level, not a single line.
